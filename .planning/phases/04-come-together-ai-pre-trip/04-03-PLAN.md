**Layout structure (per D-01, D-04):**
The page is a full-height vertical flow showing one Q&A exchange at a time, NOT a scrolling chat log. Structure:

```
[Back to journey link]
[Title: "Come Together"]
[Subtitle: "A conversation before your journey"]
[Thin progress bar: currentQuestion / 10]

[Current exchange area - centered vertically]:
  [If previous answer exists: user's last answer in muted text above]
  [AI question in large, warm text]
  [Text input area below]
  [Send button]

[Loading state when waiting for AI]
[Intention display when complete]
```

**Component structure:**
Create the following inline components within the file (per project convention — route files contain their own sub-components):

1. **ProgressBar** — thin bar at top, `currentQuestion / 10` width, accent-warm color, rounded-full, 3px height. Animate width with CSS transition (transform, per compositor-friendly rules).

2. **LoadingIndicator** — three dots pulsing gently (opacity animation). Text: "Reflecting..." in text-secondary. Use CSS `@keyframes pulse` with opacity only. Respect `prefers-reduced-motion` — if reduced motion, show static "Thinking..." text instead.

3. **ConversationView** — the main Q&A area:
   - If `isLoading`: show LoadingIndicator centered
   - If `isComplete`: show IntentionDisplay
   - Otherwise: show the current exchange
     - Find the latest assistant message (the current question)
     - Show it in large text (text-xl or text-2xl), text-text-primary, max-w-2xl, centered
     - Below: the textarea for user response
     - Below: send button

4. **UserInput** — textarea + send button:
   - `<textarea>` with dark luxury styling: bg-surface, border-2 border-card, rounded-[16px], text-text-primary, min-h-[120px], resize-none
   - Placeholder: "Take your time..." in text-secondary
   - Send button: rounded-[16px], bg-accent-warm, text-background, "Continue" label
   - Disable button while isLoading or empty input
   - Submit on Enter (but Shift+Enter for newline)
   - After submit, clear the textarea

5. **IntentionDisplay** (per D-16):
   - Show when `isComplete && intentionSentence`
   - Large text (text-3xl or text-4xl), accent-warm color, centered, max-w-xl
   - Italic style for the sentence itself
   - Above: small label "Your intention:" in text-secondary
   - Below: a subtle "This sentence was distilled from your conversation" note
   - Below that: Link back to journey overview

6. **CompletedState** — if user returns to a completed conversation:
   - Show the IntentionDisplay immediately (no conversation replay)
   - Small link: "View conversation" that could expand to show all Q&A pairs (optional, Claude's discretion)

**Fade transitions (per D-04):**
When a new AI question appears, use a CSS transition. Wrap the exchange area in a container with:
- `opacity: 0` initially, then `opacity: 1` after a short delay
- `transform: translateY(8px)` to `translateY(0)`
- `transition: opacity 400ms ease-out, transform 400ms ease-out`
- Use a `key` prop tied to `currentQuestion` to trigger remount/re-animation
- If `prefers-reduced-motion`: skip the transform, keep only opacity or no animation

**Error handling:**
- If `error` is set, show it below the input area in text-danger
- Allow retry (the sendMessage function can be called again)

**Dark luxury styling notes:**
- Background: the page inherits bg-background from root layout
- All containers use rounded-[20px] or rounded-[16px] per DSGN-03
- Generous padding (p-8 or p-12 on the main container)
- The overall feel should be spacious, calm, intentional — lots of whitespace
- No visible borders on the main container, subtle on input areas only

**Accessibility:**
- textarea has `aria-label="Your response"`
- Loading state has `aria-live="polite"` region
- Progress bar has `role="progressbar"` with `aria-valuenow` and `aria-valuemax`
- Send button is `type="submit"` within a form element
- Focus moves to textarea after AI question appears

Use double quotes and semicolons (per `src/routes/` convention).
