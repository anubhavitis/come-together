Create `src/hooks/use-conversation.ts` — the central hook for the AI conversation lifecycle.

**Hook signature:**
```typescript
export function useConversation(journeyId: string): {
  messages: ConversationMessage[]
  currentQuestion: number       // 1-10 (or 11 for intention generation)
  isLoading: boolean            // true while waiting for AI
  isComplete: boolean           // true after intention generated
  intentionSentence: string | null
  error: string | null
  sendMessage: (text: string) => Promise<void>
}
```

**Internal state:**
- `messages`: `ConversationMessage[]` — initialized from Phase1 data on mount
- `isLoading`: boolean — true during API call
- `isComplete`: boolean — true when completedAt is set
- `intentionSentence`: string | null — from `intentions.primary`
- `error`: string | null

**Initialization (resume logic, per D-13):**
- Use `usePhase1(journeyId)` to load existing data
- If `phase1?.conversation` has messages, restore them into state
- Calculate `currentQuestion` from the number of assistant messages + 1
- Use the `initialized` guard pattern (per project convention)
- If conversation is empty (new journey), trigger the first AI question automatically (the AI opens the conversation, not the user)

**sendMessage flow (per D-09, D-14):**
1. Create user message: `{ role: 'user', content: text, questionNumber: currentQuestion }`
2. Add to messages state
3. Set `isLoading = true`
4. Get auth token: `const { data: { session } } = await supabase.auth.getSession()`
5. Call `/api/chat` with:
   - `method: 'POST'`
   - `headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer ' + session.access_token }`
   - `body: JSON.stringify({ messages: messagesForApi, phase: 'phase1' })`
   - `messagesForApi` = conversation mapped to `{ role, content }` only (strip scores/questionNumber)
6. Parse response JSON: `{ message }` or `{ error }`
7. If error, set error state and return
8. Extract scores: `const { swemwbs } = parseScoresFromResponse(message)`
9. Strip scores for display: `const cleanContent = stripScoresFromResponse(message)`
10. Create assistant message: `{ role: 'assistant', content: cleanContent, questionNumber: currentQuestion, scores: swemwbs }`
11. Add to messages state
12. Set `isLoading = false`
13. Persist via `useUpsertPhase1`: save conversation array to Supabase (per D-14 — discrete save, not debounced)

**After 10 Q&A pairs (per D-15, D-16, D-17, D-18):**
When `currentQuestion > 10` after the 10th user answer:
1. Make a FINAL API call with conversation + extra system instruction appended as a user message: "Based on our entire conversation, distill one single sentence — a personal intention this person can carry into their experience. Respond with ONLY that sentence, nothing else. Also include your final SWEMWBS scoring."
2. Parse the intention from the response (strip scores)
3. Aggregate all SWEMWBS scores using `aggregateSwemwbsScores(allMessages)`
4. Save to Phase1: `intentions.primary = intentionSentence`, `swemwbs = aggregatedScores`, `completedAt = new Date().toISOString()`
5. Set `isComplete = true`, `intentionSentence` state

**First question trigger:**
When conversation is empty and data is loaded (initialized), automatically call `/api/chat` with just the system prompt context (empty messages array or a single system-level nudge). The first AI message should be the opening question. Use a `useEffect` that fires once when `initialized && messages.length === 0 && !isComplete`.

No semicolons in this file (follows `src/hooks/` convention). Single quotes.
