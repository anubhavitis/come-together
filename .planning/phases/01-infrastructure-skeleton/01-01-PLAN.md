Add `server.proxy` configuration to `vite.config.ts` (per D-07). The proxy forwards all `/api/*` requests to `vercel dev` running on port 3001. No `rewrite` is needed because the path `/api/chat` maps directly to the Vercel function at `api/chat.ts`.

Update `vite.config.ts` to this exact content:
```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import tailwindcss from "@tailwindcss/vite";
import { TanStackRouterVite } from "@tanstack/router-vite-plugin";
import path from "path";

export default defineConfig({
  plugins: [react(), tailwindcss(), TanStackRouterVite()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
  server: {
    proxy: {
      "/api": {
        target: "http://localhost:3001",
        changeOrigin: true,
      },
    },
  },
});
```

Key details:
- `"/api"` key matches all paths starting with `/api` (including `/api/chat`)
- `target` points to port 3001 where `vercel dev --listen 3001` will run
- `changeOrigin: true` sets the Host header to match the target
- No `rewrite` property -- path is preserved as-is
