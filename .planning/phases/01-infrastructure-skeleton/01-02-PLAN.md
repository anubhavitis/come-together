Structure (in order):
1. **Imports**: `createClient` from `@supabase/supabase-js`, `generateText` from `ai`, `anthropic` from `@ai-sdk/anthropic`. Use `import type` for type-only imports per project convention.

2. **Env var validation** (module-level, follows `src/lib/supabase.ts` pattern): Read `process.env.SUPABASE_URL`, `process.env.SUPABASE_SERVICE_ROLE_KEY`, `process.env.ANTHROPIC_API_KEY`. This creates a service_role client that bypasses RLS -- used only for auth verification, not data queries.

4. **Model constant**: `const MODEL_ID = 'claude-3-5-haiku-20241022'` (per research recommendation for better quality at low cost; trivially changeable).

5. **System prompt constant**: A placeholder string for the Phase 1 skeleton. Content: `'You are a warm, thoughtful guide helping someone prepare for a meaningful personal experience. Respond with empathy and insight. Keep responses concise but genuine.'` (Claude's discretion per CONTEXT.md).

6. **`verifyUser` helper function**: Extracts Bearer token from `Authorization` header. Calls `supabaseAdmin.auth.getUser(token)`. Returns the user object on success, `null` on failure (auth error or no user).
   ```
   async function verifyUser(request: Request)
   ```
   - Get `authorization` header from request
   - If missing or doesn't start with `'Bearer '`, return null
   - Extract token: `authHeader.slice(7)`
   - Call `supabaseAdmin.auth.getUser(token)`
   - If error or no user, return null
   - Return user

7. **`export async function POST(request: Request)`**: The named export handler per Vercel Web API pattern (D-02).
   Flow:
   a. Call `verifyUser(request)` -- if null, return `Response.json({ error: 'Missing or invalid authorization token' }, { status: 401 })`
   b. Parse body: `const { messages } = await request.json()` -- the messages array per D-06
   c. Wrap AI call in try/catch:
      - Call `generateText({ model: anthropic(MODEL_ID), system: SYSTEM_PROMPT, messages })`
      - Return `Response.json({ message: text })` with the destructured `text` from result
   d. Catch block: narrow error with `err instanceof Error`, return `Response.json({ error: message }, { status: 500 })`

Key constraints:
- No semicolons (matches src/lib/ style)
- Single quotes (matches src/lib/ style)
- `import type` for type-only imports
- No `console.log` in production code
- No streaming -- `generateText` returns the full response (per D-01, D-05)
