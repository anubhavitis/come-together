  <action>
Create `src/components/shared/question-carousel.tsx` implementing the Typeform-style carousel. Follow existing component conventions: plain function declaration, no React.FC, PascalCase component name, props interface named `QuestionCarouselProps`.

**Exported types:**
```typescript
export interface CarouselQuestion {
  id: string
  content: React.ReactNode  // Render prop - caller provides the question UI
}

export interface QuestionCarouselProps {
  questions: CarouselQuestion[]
  currentIndex: number
  onAdvance: () => void
  canAdvance?: boolean        // Whether Enter should trigger advance (default: false)
  totalQuestions?: number     // For progress bar display (may differ from questions.length for streaming)
  className?: string
}
```

**Component structure (per D-10, D-11, D-12, D-13, D-14, D-15):**

1. **Progress bar** at top: A thin (2px height, `h-0.5`) line. Background: `bg-card`. Filled portion: `bg-accent-warm`. Width calculated as `((currentIndex + 1) / total) * 100%`. Animate width with `transition-[width] duration-300` (CSS transition on width property). Add `role="progressbar"`, `aria-valuenow={currentIndex + 1}`, `aria-valuemin={1}`, `aria-valuemax={total}`, `aria-label={`Question ${currentIndex + 1} of ${total}`}`.

2. **Question container**: `flex flex-1 items-center justify-center px-4 py-8`. This centers the question content vertically and horizontally in remaining viewport space. The container should use `min-h-[calc(100dvh-56px)]` to fill viewport minus header height (56px = h-14).

3. **Transition wrapper** around `questions[currentIndex].content`:
   - Use a `key={questions[currentIndex].id}` on the wrapper div to trigger React re-mount on question change.
   - When `useReducedMotion()` returns `false` (motion OK): apply CSS transition classes. Use inline styles for the transition:
     - Initial render state (controlled by local useState): `opacity: 0, transform: translateY(8px)`
     - After mount (via useEffect with short requestAnimationFrame): `opacity: 1, transform: translateY(0)`
     - Transition: `transition: opacity 300ms ease-out, transform 300ms ease-out`
   - When `useReducedMotion()` returns `true`: no transition styles at all, instant display.
   - Import `useReducedMotion` from `@/hooks`.

4. **Focus management** (per D-14, CRSL-05): 
   - Add a `ref` to the question container div with `tabIndex={-1}`.
   - In a `useEffect` triggered by `currentIndex` change, call `questionRef.current?.focus()` to move focus to the new question.
   - This ensures keyboard users and screen readers are oriented to the new content.

5. **Screen reader announcement** (per D-14, CRSL-05):
   - Add a visually-hidden div with `aria-live="polite"` and class `sr-only` (Tailwind's screen-reader-only class).
   - Content: `Question ${currentIndex + 1} of ${total}`.
   - This is SEPARATE from the question content to avoid double-announcement (per RESEARCH pitfall #6).

6. **Keyboard handler** (per D-13, CRSL-04):
   - In a `useEffect`, add a `keydown` event listener on `document`.
   - **Enter key**: If `e.key === 'Enter'` and `canAdvance` is true and `e.target` is NOT a textarea: call `e.preventDefault()` then `onAdvance()`.
   - **ArrowUp/ArrowDown keys**: When pressed, find all focusable option elements within the question container (`questionRef.current`). Query for `[role="radio"], [role="option"], button[data-option]` selectors within the container. If matches exist:
     - ArrowDown: move focus to the next option in the list (wrap to first if at end).
     - ArrowUp: move focus to the previous option (wrap to last if at start).
     - Call `e.preventDefault()` to prevent page scroll.
   - This allows the carousel to handle option cycling for any render prop content that uses standard ARIA roles or `data-option` attributes on its choice buttons. Phase 3 callers render buttons with `role="radio"` or `data-option`, and the carousel handles the keyboard cycling.
   - Clean up the listener on unmount.

7. **Mobile layout** (per D-15, CRSL-06):
   - The outer container uses `min-h-[calc(100dvh-56px)]` with `flex flex-col`.
   - All spacing uses padding, not margin, for consistent box model.
   - The component itself does not render interactive elements with touch targets (those come from the render prop content), but the layout ensures content is centered and accessible.

**Style conventions:**
- Use double quotes for JSX string attributes (matches src/components/ convention)
- Use semicolons (matches src/components/ convention)
- Use Tailwind utility classes inline, no clsx/cn helpers
- Import `{ useState, useEffect, useRef, useCallback }` from 'react'
- Import `{ useReducedMotion }` from '@/hooks'
