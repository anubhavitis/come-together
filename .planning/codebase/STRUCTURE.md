## Directory Purposes

**`src/routes/`:**
- Purpose: All page-level components, one file per URL segment
- Contains: Route definitions via `createFileRoute()`, page components, inline sub-components specific to that page
- Key files: `__root.tsx` (layout + auth guard), `index.tsx` (journey list), `journey/$id/phase1.tsx` (largest form, ~450 lines)
- Pattern: Route file exports `const Route = createFileRoute(path)({ component })`. Page component and any helpers co-located in the same file.

**`src/hooks/`:**
- Purpose: All data fetching and mutation logic, isolated from UI
- Contains: One file per data domain + two cross-cutting hooks (`use-auth`, `use-auto-save`)
- Key files: `use-journeys.ts` (contains all journey mutations + FullJourney query), `use-auto-save.ts` (generic, not domain-specific)
- Pattern: Each hook file exports named hooks prefixed with `use`. Barrel `index.ts` re-exports all.

**`src/components/shared/`:**
- Purpose: Instrument input primitives used across all phase forms
- Contains: Six UI components, barrel `index.ts`
- Key files: `likert-scale.tsx`, `rating-slider.tsx`, `vas-slider.tsx`
- Note: No instrument-specific composite components yet (e.g., no `MEQ30Form` component — instruments are assembled inline in phase route files)

**`src/lib/`:**
- Purpose: Infrastructure and domain-agnostic utilities
- Contains: Supabase client, Zod validation schemas, JSON export/import
- Key files: `supabase.ts` (validates env vars at import time; throws if missing), `schemas.ts` (Zod schemas for all JSONB columns)

**`src/types/`:**
- Purpose: Single source of truth for domain TypeScript types
- Contains: All journey-related types in one file
- Key file: `src/types/journey.ts` — defines `Journey`, `Phase1`, `Phase2`, `Phase3Entry`, `FullJourney`, and all instrument subtypes

**`src/data/`:**
- Purpose: Static content — instrument question text and IDs
- Contains: `swemwbs-items.ts` (typed `as const` array)
- Note: Only SWEMWBS items are here; MEQ30, EDI, EBI item text is not yet extracted to data files

**`supabase/migrations/`:**
- Purpose: Database schema definitions
- Contains: `001_initial_schema.sql` — creates `journeys`, `phase1`, `phase2`, `phase3_entries` with RLS and `updated_at` triggers
- Generated: No — hand-authored
- Committed: Yes

## Key File Locations

**Entry Points:**
- `src/main.tsx`: App bootstrap — QueryClient config, router initialization
- `index.html`: Vite HTML root
- `src/routes/__root.tsx`: Global layout, auth redirect guard

**Configuration:**
- `vite.config.ts`: Plugins (React, Tailwind, TanStackRouter), `@` alias to `src/`
- `src/index.css`: Tailwind v4 `@theme` block with all design tokens
- `tsconfig.app.json`: TypeScript settings for application code
- `supabase/migrations/001_initial_schema.sql`: Full DB schema

**Core Logic:**
- `src/hooks/use-journeys.ts`: Journey CRUD, FullJourney join query, row mappers
- `src/hooks/use-auto-save.ts`: Debounce + flush auto-save logic
- `src/lib/supabase.ts`: Single Supabase client instance
- `src/lib/schemas.ts`: Zod validation for all instrument JSONB data
- `src/types/journey.ts`: All domain types

**Auto-generated (do not edit):**
- `src/routeTree.gen.ts`: Generated by TanStack Router Vite plugin on every dev/build run

## Naming Conventions

**Files:**
- Routes: match URL path segments, kebab-case — `phase1.tsx`, `phase3.new.tsx`, `phase3.$entryId.tsx`
- Hooks: `use-` prefix, kebab-case — `use-auto-save.ts`, `use-phase1.ts`
- Components: kebab-case — `likert-scale.tsx`, `collapsible-section.tsx`
- Types/lib: kebab-case — `schemas.ts`, `supabase.ts`

**Directories:**
- Dynamic route segments use TanStack Router `$param` syntax: `$id/`, `$entryId`
- Components grouped by role: `shared/` for cross-phase primitives

**Functions and Components:**
- Components: PascalCase — `LikertScale`, `Phase1Form`, `JourneyCard`
- Hooks: camelCase with `use` prefix — `usePhase1`, `useAutoSave`, `useUpsertPhase1`
- Mapper functions: `mapPhase1`, `mapPhase2`, `mapPhase3Entry`, `mapJourney` (private to hook files)
- Snake case conversion helpers: `toSnake` (private to each phase hook file)

**Query Keys:**
- List: `['journeys']`
- Single journey (with phases): `['journey', id]`
- Phase data: `['phase1', journeyId]`, `['phase2', journeyId]`, `['phase3', journeyId]`
- Single phase3 entry: `['phase3-entry', entryId]`

## Where to Add New Code

**New phase form page:**
- Implementation: `src/routes/journey/$id/phase{N}.tsx`
- Hook: `src/hooks/use-phase{N}.ts` with `mapPhase`, `toSnake`, query, and upsert mutation
- Export hook from: `src/hooks/index.ts`
- Types: Add to `src/types/journey.ts`
- Zod schemas: Add to `src/lib/schemas.ts`

**New shared UI component:**
- Implementation: `src/components/shared/{component-name}.tsx`
- Export from: `src/components/shared/index.ts`

**New instrument item data:**
- Implementation: `src/data/{instrument-name}-items.ts` as a `const` array with `{ id, text }` shape
- Follow pattern of `src/data/swemwbs-items.ts`

**New utility function:**
- Place in: `src/lib/{utility-name}.ts`
- If it involves Supabase calls without query caching, put in `src/lib/` (like `export.ts`)
- If it wraps TanStack Query, put in `src/hooks/`

**New route/page:**
- Create file at: `src/routes/{path}.tsx`
- TanStack Router will auto-update `src/routeTree.gen.ts` on next `bun dev`
- Export `const Route = createFileRoute('/your/path')({ component: YourComponent })`

## Special Directories

- Purpose: Auto-generated static route tree consumed by `createRouter` in `main.tsx`
- Generated: Yes — by `TanStackRouterVite` plugin
- Committed: Yes (required for TypeScript type inference)

**`.planning/`:**
- Purpose: GSD planning documents
- Generated: No
- Committed: Yes

**`supabase/migrations/`:**
- Purpose: Postgres schema migrations applied to Supabase project
- Generated: No
- Committed: Yes
